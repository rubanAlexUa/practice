# Завдання №3: Спадкування

Згідно цього завдання потрібно було:

 - Як основа використовувати вихідний текст проекту попередньої лабораторної роботи. Забезпечити розміщення результатів обчислень уколекції з можливістю збереження/відновлення.
 -  Використовуючи шаблон проектування Factory Method (Virtual Constructor), розробити ієрархію, що передбачає розширення рахунок додавання нових відображуваних класів.
 - Розширити ієрархію інтерфейсом "фабрикованих" об'єктів, що представляє набір методів для відображення результатів обчислень.
 - Реалізувати ці методи виведення результатів у текстовому виде.
 - Розробити тареалізувати інтерфейс для "фабрикуючого" методу.

## Пояснення структури коду

Для реалізації завданнябуло взято код із минулої роботи, та змінено із внесенням таких змін: 

Додав до класу Item метод recalculate() який рахує опір заново, це потрібно після того як дані зчитуються з файлу.

Клас Clac переписав, і тепер замість одного результату зберігається список всіх обчислень через ArrayList.
Також додав збереження і зчитування даних з текстового файлу через save() і restore(). Крім того тепер Calc реалізує  
інтерфейс ViewFactory, тобто він є фабрикою.

Для реалізації Factory Method було додано три нових класи. Інтерфейс ViewFactory який  
оголошує фабричний метод createView(). Інтерфейс ItemView який оголошує метод show() — це правило  
для всіх класів які виводять результати. TextItemView — це клас який реалізує ItemView і виводить результати в консоль у текстовому вигляді. 

Клас Main залишився без змін окрім додавання команди s для збереження даних у файл.
